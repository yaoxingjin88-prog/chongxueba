# AI电子宠物自律养成平台四个成长模型设计方案

## 1. 设计目标

本项目是一个面向学生用户的 **AI电子宠物自律养成平台**。为了让用户在学习过程中获得持续反馈和成长激励，系统需要设计一套可视化、可量化、可养成的成长模型。

四个成长模型主要用于：

- 记录用户学习行为
- 驱动宠物成长与进化
- 提供首页可视化展示
- 增强用户长期使用动力
- 支撑成长报告、任务中心、成就系统等功能

首页可以围绕一个 3D 宠物形象，将四个成长模型以能量值、等级、进度条、徽章、场景变化等形式展示出来。

------

# 2. 四个成长模型设计

## 2.1 专注成长模型

### 模型定位

专注成长模型用于衡量用户的学习专注能力，主要反映用户在学习过程中的专注时长、专注稳定性和任务完成情况。

### 核心数据

| 数据项       | 说明                           |
| ------------ | ------------------------------ |
| 今日专注时长 | 用户当天累计专注学习时间       |
| 累计专注时长 | 用户历史总专注时间             |
| 单次最长专注 | 用户单次学习的最长专注时间     |
| 连续专注天数 | 用户连续完成专注任务的天数     |
| 专注完成率   | 用户设定专注目标后的完成比例   |
| 中断次数     | 学习过程中提前退出或暂停的次数 |

### 成长规则

| 等级  | 名称       | 达成条件          |
| ----- | ---------- | ----------------- |
| Lv.1  | 初识专注   | 累计专注 1 小时   |
| Lv.5  | 稳定专注   | 连续专注 3 天     |
| Lv.10 | 专注达人   | 累计专注 50 小时  |
| Lv.20 | 深度专注者 | 连续专注 21 天    |
| Lv.30 | 星级专注者 | 累计专注 200 小时 |

### 首页展示方式

首页可以将专注成长模型设计为宠物周围的 **专注能量环**。

表现方式：

- 专注值越高，能量环越亮
- 连续专注天数越多，光环层数越多
- 学习中断次数较多时，光环变弱
- 用户进入专注模式时，宠物可以进入读书、冥想、闭眼专注等状态

### 3D表现建议

| 专注状态     | 3D视觉表现                   |
| ------------ | ---------------------------- |
| 专注值低     | 光环较暗，宠物表情普通       |
| 专注值中等   | 宠物进入读书状态，周围有星光 |
| 专注值高     | 宠物周围出现稳定能量环       |
| 连续专注优秀 | 宠物身后出现专注星轨或魔法阵 |

------

## 2.2 习惯养成模型

### 模型定位

习惯养成模型用于衡量用户是否形成稳定的学习习惯，重点关注每日打卡、任务完成、固定学习时间等行为。

### 核心数据

| 数据项           | 说明                         |
| ---------------- | ---------------------------- |
| 每日打卡         | 用户每天是否完成学习打卡     |
| 连续打卡天数     | 用户连续完成打卡的天数       |
| 学习任务完成数   | 用户每日完成任务数量         |
| 习惯完成率       | 用户计划任务的完成比例       |
| 固定时间学习次数 | 用户是否在固定时间段进行学习 |
| 断签次数         | 用户中断习惯养成的次数       |

### 成长规则

| 等级  | 名称     | 达成条件       |
| ----- | -------- | -------------- |
| Lv.1  | 习惯萌芽 | 连续打卡 1 天  |
| Lv.5  | 习惯幼苗 | 连续打卡 7 天  |
| Lv.10 | 习惯小树 | 连续打卡 21 天 |
| Lv.20 | 自律花园 | 连续打卡 45 天 |
| Lv.30 | 自律森林 | 连续打卡 60 天 |

### 首页展示方式

首页可以将习惯养成模型设计为宠物脚下的 **成长小岛场景**。

表现方式：

- 用户连续打卡，小岛会长出花草
- 学习习惯越稳定，场景越丰富
- 连续打卡越久，小岛面积越大
- 断签后，部分植物会变暗或进入恢复状态

### 3D表现建议

| 习惯状态       | 3D视觉表现           |
| -------------- | -------------------- |
| 刚开始养成     | 小岛上只有草地       |
| 连续打卡 7 天  | 出现小花、小树       |
| 连续打卡 21 天 | 出现学习石碑、星星灯 |
| 连续打卡 60 天 | 形成完整自律花园     |

## 2.3 宠物进化模型

### 模型定位

宠物进化模型是整个项目最核心的成长模型，用于驱动电子宠物的等级、经验、形态和外观变化。

用户完成学习任务、专注计时、打卡、互动等行为后，可以获得经验值，推动宠物升级和进化。

### 核心数据

| 数据项       | 说明                           |
| ------------ | ------------------------------ |
| 宠物等级     | 当前宠物等级                   |
| 宠物经验值   | 当前累计经验                   |
| 下一等级经验 | 升级所需经验                   |
| 宠物阶段     | 当前宠物成长阶段               |
| 喂养次数     | 用户与宠物互动次数             |
| 学习贡献值   | 学习行为带来的成长值           |
| 互动次数     | 用户摸头、喂养、换装等操作次数 |

### 进化阶段

| 阶段   | 形态   | 达成条件       |
| ------ | ------ | -------------- |
| 阶段 1 | 宠物蛋 | 新用户注册     |
| 阶段 2 | 幼年体 | 宠物达到 Lv.5  |
| 阶段 3 | 成长期 | 宠物达到 Lv.15 |
| 阶段 4 | 进化体 | 宠物达到 Lv.30 |
| 阶段 5 | 星耀体 | 宠物达到 Lv.50 |

### 经验获取方式

| 行为             | 获得经验 |
| ---------------- | -------- |
| 完成一次专注学习 | +60 EXP  |
| 完成每日打卡     | +30 EXP  |
| 完成学习任务     | +50 EXP  |
| 连续学习 7 天    | +200 EXP |
| 获得新成就       | +100 EXP |
| 与宠物互动       | +10 EXP  |

### 首页展示方式

首页中间展示一个 3D 宠物模型，宠物的外观、动作、光效会随着等级变化。

展示内容包括：

- 宠物名称
- 宠物等级
- 当前经验值
- 下一阶段进化进度
- 当前成长阶段
- 宠物状态文案

示例文案：

```text
小橘 Lv.23
当前阶段：成长期
距离进化为「星辉狐」还差 350 EXP
```

### 3D表现建议

| 等级阶段      | 3D表现                                   |
| ------------- | ---------------------------------------- |
| Lv.1 - Lv.5   | 宠物蛋，可轻微晃动                       |
| Lv.6 - Lv.15  | 幼年体，会眨眼、摇尾巴                   |
| Lv.16 - Lv.30 | 成长期，可读书、跳跃、挥手               |
| Lv.31 - Lv.50 | 进化体，拥有光环、星星特效               |
| Lv.50 以上    | 星耀体，拥有专属服饰、悬浮粒子、动态背景 |

## 2.4 成就激励模型

### 模型定位

成就激励模型用于记录用户在学习、自律、宠物养成、社交互动等方面获得的荣誉与奖励。

该模型主要用于增强用户成就感，提升长期留存。

### 核心数据

| 数据项     | 说明                   |
| ---------- | ---------------------- |
| 徽章数量   | 用户已获得的徽章数量   |
| 成就等级   | 用户当前成就成长等级   |
| 完成挑战数 | 用户完成的挑战任务数量 |
| 排名变化   | 用户在排行榜中的变化   |
| 奖励领取数 | 用户领取奖励的次数     |
| 稀有成就数 | 用户获得的高级成就数量 |

### 成就类型

| 类型     | 示例                                       |
| -------- | ------------------------------------------ |
| 专注成就 | 首次专注、连续专注 7 天、累计专注 100 小时 |
| 习惯成就 | 连续打卡、早起学习、夜间学习               |
| 宠物成就 | 第一次进化、满级宠物、获得稀有皮肤         |
| 任务成就 | 完成每日任务、完成阶段挑战                 |
| 社交成就 | 好友助力、排行榜前三、组队学习             |

### 首页展示方式

首页可以将成就激励模型设计为宠物背后的 **3D徽章墙** 或 **星座成就环**。

表现方式：

- 成就越多，宠物背后的徽章越丰富
- 稀有成就可以显示特殊光效
- 最近获得的成就可以浮动展示
- 点击徽章可进入成就系统页面

### 3D表现建议

| 成就状态     | 3D视觉表现                   |
| ------------ | ---------------------------- |
| 成就较少     | 显示少量星星徽章             |
| 成就中等     | 宠物背后出现徽章环           |
| 成就较多     | 出现完整徽章墙               |
| 获得稀有成就 | 出现金色光效、奖杯、烟花粒子 |

# 4. 3D实现方案

## 4.1 方案一：轻量级真实 3D

### 技术方案

- Vue3
- Three.js
- GLB / glTF 模型
- Blender 建模
- CSS 动效
- 后端返回模型地址与成长数据

### 实现流程

```text
Blender 制作宠物模型
↓
导出 .glb 文件
↓
Vue3 首页使用 Three.js 加载模型
↓
根据用户等级切换模型、动作、光效
↓
用户点击宠物触发互动动画
```

### 优点

- 视觉效果好
- 可以真实旋转
- 适合 H5 和 Web 展示
- 比较适合比赛项目展示

### 缺点

- 需要准备 3D 模型
- 模型文件过大会影响加载速度
- 需要一定 Three.js 基础

------

## 4.2 方案二：伪 3D 动效

### 技术方案

- 2D PNG 图片
- Lottie 动画
- Spine 动画
- CSS 视差效果
- 渐变背景与粒子动画

### 实现方式

虽然不是真 3D，但可以通过视觉设计模拟 3D 效果：

- 宠物使用透明 PNG
- 宠物上下浮动
- 背景云朵缓慢移动
- 星星粒子闪烁
- 点击宠物播放眨眼、跳跃动画
- 根据等级替换不同宠物图片

### 优点

- 开发简单
- 性能好
- 移动端流畅
- 不需要复杂建模

### 缺点

- 不能真正 360 度旋转
- 交互感弱于真实 3D

------

## 4.3 方案三：高级 3D 互动

### 技术方案

- Vue3
- Three.js
- Blender
- Mixamo 骨骼动画
- WebGL 粒子系统
- 3D 场景换装系统

### 可实现功能

- 宠物待机动画
- 摸头互动
- 学习时专注动画
- 升级进化动画
- 换装系统
- 3D场景装饰
- 徽章墙展示
- 宠物状态切换

### 适用阶段

该方案适合项目后期产品化，或者用于高级版本展示。

------

# 5. 推荐实现方案

结合当前项目的设计风格和开发成本，建议采用：

> 伪 3D + 局部 Three.js 的混合方案

具体做法：

- 宠物主体使用 3D GLB 模型
- 背景、星星、云朵、光效使用 CSS 或 Lottie
- 成长模型用卡片展示
- 宠物支持点击互动
- 等级变化时切换宠物模型或皮肤
- 成就变化时展示徽章光效

这样既能体现 3D 质感，又不会让开发难度过高。

# 6. 四个成长模型与 3D视觉联动

| 成长模型     | 数据变化     | 3D表现           |

| 专注成长模型 | 专注时长越高 | 宠物身边光环越亮 |

```text
成就变多：
宠物背后出现星星徽章
```

# 7. 数据库设计

## 7.1 成长模型表

```sql
CREATE TABLE growth_model (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  model_type VARCHAR(50) NOT NULL COMMENT 'focus/habit/pet/achievement',
  level INT DEFAULT 1,
  exp INT DEFAULT 0,
  score INT DEFAULT 0,
  stage VARCHAR(50),
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

字段说明：

| 字段       | 说明         |
| ---------- | ------------ |
| id         | 主键         |
| user_id    | 用户 ID      |
| model_type | 成长模型类型 |
| level      | 当前模型等级 |
| exp        | 当前经验值   |
| score      | 当前成长分数 |
| stage      | 当前成长阶段 |
| updated_at | 更新时间     |

------

```sql
  exp INT DEFAULT 0,
  stage VARCHAR(50),
  model_url VARCHAR(255),
```

字段说明：

| 字段       | 说明           |
| ---------- | -------------- |
| id         | 主键           |
| user_id    | 用户 ID        |
| pet_name   | 宠物名称       |
| pet_type   | 宠物类型       |
| level      | 宠物等级       |
| exp        | 当前经验值     |

# 10. 总结

本项目的四个成长模型可以设计为：

| 模型         | 作用                         | 首页表现        |
| ------------ | ---------------------------- | --------------- |
| 专注成长模型 | 衡量用户学习专注能力         | 专注能量环      |
| 习惯养成模型 | 衡量用户自律习惯稳定性       | 成长小岛        |
| 宠物进化模型 | 驱动宠物等级、经验、形态变化 | 3D宠物形态      |
| 成就激励模型 | 记录用户荣誉、徽章、奖励     | 徽章墙 / 星座环 |

最终首页展示效果建议为：

> 一个 3D 宠物 + 四个成长能量入口 + 进化进度条 + 今日任务入口。

这种设计既能突出宠物养成感，也能让用户清楚看到自己的学习成长状态，比较适合作为 AI电子宠物自律养成平台的核心首页展示方案。
